# JITコンパイル入門

**実行しながらコンパイルする、という発想の地図**

プログラムは、書いたとおりに動けばよい —— けれど、**書いたとおりに動くだけでは遅い**。
インタプリタは柔軟で移植しやすい代わりに、1 命令ごとに「次は何をするのか」を
読み解く固定費を払い続けます。あらかじめ機械語へ変換しておく AOT
コンパイルは速い代わりに、動的な言語の「実際にどう使われるか」を知りません。

**JIT（Just-In-Time、実行時コンパイル）**は、この 2 つのあいだに立ちます。
プログラムを動かしながら、よく実行される部分を、**そのとき観測した事実を
前提に**機械語へコンパイルする。JavaScript が C に迫る——ときに超える——速度で動き、
Java や Ruby や Python 処理系が「インタプリタ言語なのに速い」と言われるとき、
その裏側には JIT や適応的な特殊化コンパイルがあります。

本書は、JIT コンパイルの**全体地図**を描きます。

- **第 I 部 基本のテクニック** —— なぜ速くなるのかという原理から、
  最小の JIT を実際に手で書き、メソッド JIT とトレーシング JIT、
  中間表現、プロファイリング、メタトレーシング、階層的 JIT、
  脱最適化と OSR、主要なフレームワークまでを通します。
- **第 II 部 最新研究** —— 部分評価と Futamura 射影、ウォームアップ問題と
  コンパイルキャッシュ、Copy-and-Patch、投機的最適化の正しさの形式化、
  WebAssembly の JIT を、2010〜2026 年の論文をたどって解説します。
- **第 III 部 ケーススタディ** —— JVM・V8・Ruby・Python/PyPy・LuaJIT、
  そしてその他の処理系の JIT を、言語固有の工夫とコードで紹介します。

サンプルは原理を映す**動くコード**を主役にします。多くは Ruby ですが、
機械語生成のところでは C やアセンブリ（x86-64）も使います。

本書は、言語処理系入門の副読本として読めるように書かれています。
バイトコードやインタプリタの基礎は別途
[言語処理系入門](https://kolanglab.github.io/book_langimpl_intro/) にゆずり、
ここでは「その先の速さ」を扱います。
