# ケーススタディ：LuaJIT

**LuaJIT** は、Mike Pall がほぼ一人で作り上げた、Lua 言語の処理系です。
[トレーシング JIT](tracing.md)の一つの到達点であり、その完成度の高さと
実装の美しさで、言語処理系の世界で伝説的な評価を受けています
[](#cite:luajit)。「小さな言語に、これほど洗練された JIT を載せられるのか」
という驚きが、LuaJIT にはあります。この章では、LuaJIT を LuaJIT たらしめて
いる技術を見ます。

## まず土台：手書きアセンブリのインタプリタ

LuaJIT の速さは、JIT に入る前から始まっています。LuaJIT の**インタプリタ
自体が、各 CPU 向けに手書きされたアセンブリ**で書かれているのです。
[フレームワーク](frameworks.md)の章で紹介した **DynASM**[](#cite:dynasm) ——
Mike Pall が LuaJIT のために作ったアセンブラ —— を使って、バイトコード
ディスパッチの 1 命令 1 命令が、極限までチューニングされた機械語として
実装されています。

ふつうのインタプリタは C で書かれ、C コンパイラの出力に速さを委ねます。
LuaJIT は、[なぜ JIT か](why-jit.md)で見た**ディスパッチのコスト**を、
人間がアセンブリで直接削り込みました。CPU のレジスタの使い方、分岐予測の
効きやすさ、命令キャッシュの収まりまで計算し尽くされています。この結果、
LuaJIT は**JIT が起動する前のインタプリタの段階で、すでに多くの処理系より
速い**のです。[階層的 JIT](tiered.md)でいう「下の段」が、異様に強い。

> [!NOTE]
> 「インタプリタをアセンブリで手書きする」のは、現代では珍しい選択です。
> 移植性は下がり（CPU ごとに書き直す）、保守も難しい。それでも LuaJIT が
> これを選んだのは、Lua という小さな言語ゆえに命令セットが小さく、手書きで
> 管理できる規模だったこと、そして「速さに一切妥協しない」という設計者の
> 哲学があったからです。言語の小ささが、極端な最適化を可能にした例です。

## トレーシング JIT としての LuaJIT

JIT 部分は、[トレーシング](tracing.md)の章で見た仕組みの、磨き抜かれた
実装です。ホットなループを検出すると、その 1 周を[トレース](tracing.md)
として記録し、[分岐をガードに変えた一直線](tracing.md)のコードを作ります。

LuaJIT のトレース最適化は、独自の[中間表現](ir.md)（線形の SSA 風 IR）の
上で行われます。[トレーシングの章](tracing.md)で挙げた最適化 —— ループ
不変式の巻き上げ、ガードの統合、共通部分式の削除 —— に加え、LuaJIT が
特に得意とするのが**アロケーションシンキング**（allocation sinking）です。
これは[エスケープ解析](glossary.md)の一種で、ループ内で作って同じループ内で
使い切る短命なテーブルやオブジェクトの確保を、トレースの外へ追い出したり、
そもそも作らずレジスタ上の値で済ませたりします。[JVM のスカラ置換](case-jvm.md)や
[トレーシングのアロケーション除去](tracing.md)と同じ思想を、トレース上で
徹底的に効かせています。

## Lua 固有の値表現：NaN ボクシング

LuaJIT を語るうえで外せないのが、**NaN ボクシング**（NaN boxing、
NaN タギングとも）という値表現の妙技です。[なぜ JIT か](why-jit.md)で見た
[ボクシング](why-jit.md)のコストを、Lua の言語特性を逆手に取って回避します。

Lua の数値は、伝統的にすべて**倍精度浮動小数点数（64 ビット double）**です。
ここに、数値以外の値（テーブル、文字列、関数、`nil`、真偽値）も**同じ
64 ビットに詰め込む**のが NaN ボクシングです。鍵は、IEEE 754 浮動小数点の
**NaN**（Not a Number）の表現に、膨大な「使われていないビットパターン」が
あることです。

```
通常の double          : そのまま数値として解釈
NaN の空きビットパターン : 下位ビットに「型タグ＋ポインタ」を詰め込む
  ┌──────────────┬──────────────────────────────┐
  │ NaN を示す上位ビット │ 型タグ + テーブル等へのポインタ      │
  └──────────────┴──────────────────────────────┘
```

つまり、

- ビット列が**正常な double**なら、それは Lua の数値そのもの。
- ビット列が**特定の NaN パターン**なら、空きビットに埋め込まれた
  タグとポインタを読んで、テーブルや文字列だと解釈する。

この表現の利点は絶大です。**数値は箱に入れる必要がまったくない**
（64 ビットの値そのものが数値）。[ボクシングのコスト](why-jit.md)が、
数値計算において完全に消えます。Lua が数値計算で猛烈に速いのは、
JIT の力に加えて、この値表現がもともと数値に有利だからです。トレースが
数値演算を特殊化するとき、アンボックス／ボックスの往復が要らないので、
生成コードがとても素直になります。

> [!IMPORTANT]
> NaN ボクシングは、「**言語の性質（Lua の数値は全部 double）を、値表現の
> 設計に活かす**」見事な例です。本書でくり返し見てきたとおり、JIT の
> 速さはデータ構造・値表現と一体です。[V8 のヒドゥンクラス](case-v8.md)、
> [Ruby の Object Shapes と即値](case-ruby.md)、そして LuaJIT の NaN
> ボクシング —— どれも「JIT が特殊化しやすいように値や形を表現する」
> という同じ問題への、言語ごとの答えです。

## もう一つの名物：FFI

LuaJIT には、性能とは別の角度で有名な機能があります。**FFI**（Foreign
Function Interface、外部関数インタフェース）ライブラリです。これは、
C のライブラリや構造体を、**Lua のコードから直接、宣言だけで呼べる**
ようにするものです。

```lua
local ffi = require("ffi")
ffi.cdef[[
  int printf(const char *fmt, ...);   -- C の関数を宣言するだけ
]]
ffi.C.printf("Hello %d\n", 42)        -- Lua から直接呼べる
```

ふつう、スクリプト言語から C を呼ぶには、両者をつなぐ「グルーコード」を
C で書く必要があります。LuaJIT の FFI は、C の宣言を読むだけでその面倒を
なくします。そして JIT との連携が秀逸で、**FFI 経由の C 呼び出しや C の
構造体アクセスを、トレースの中に[インライン化](why-jit.md)して機械語へ
畳み込めます**。Lua のコードと C のデータアクセスが、トレース上でひと続きの
高速な機械語になる。スクリプト言語でありながら、C 並みの低レベル操作を
JIT の速度で行える —— これが LuaJIT を、ゲームエンジンや高性能な組み込み
用途で人気にした大きな理由です。

## なぜケーススタディとして重要か

LuaJIT が本書で重要なのは、**第 I 部・第 II 部で見た技法が、一つの処理系の
中で美しく統合されている**からです。

- 手書きアセンブリの強いインタプリタ（[ディスパッチ削減](why-jit.md)の極北）
- [トレーシング JIT](tracing.md)とその[ガード・サイドイグジット](tracing.md)
- [SSA 風中間表現](ir.md)上の最適化と[アロケーション除去](tracing.md)
- 言語特性を活かした値表現（NaN ボクシング）
- JIT と[低レベルアクセス（FFI）](frameworks.md)の融合

そして、これらを**ほぼ一人の設計者が、一貫した美学で**作り上げたことが、
LuaJIT を特別な教材にしています。「小さな言語に、本書の技法を全部詰め込むと
どうなるか」の答えが、ここにあります。

## まとめ

- **LuaJIT**[](#cite:luajit)は、Mike Pall がほぼ一人で作った Lua の
  [トレーシング JIT](tracing.md)の到達点。
- インタプリタ自体を **DynASM** で**手書きアセンブリ**化し、JIT 前から
  異様に速い。言語の小ささが極端な最適化を可能にした。
- トレース最適化は独自の[SSA 風 IR](ir.md)上で行い、**アロケーション
  シンキング**（[エスケープ解析](glossary.md)）に強い。
- **NaN ボクシング**で、Lua の数値（全部 double）に他の型を相乗りさせ、
  数値の[ボクシング](why-jit.md)を完全に消す。言語特性を値表現に活かす好例。
- **FFI** で C を宣言だけで呼べ、その C アクセスをトレースに[インライン
  化](why-jit.md)できる。スクリプト言語で C 並みの低レベル操作を JIT 速度で。

最後の章では、ここまで扱いきれなかった処理系たち —— JavaScriptCore・
.NET・Julia・Dart・ART・Self —— で、地図の余白を埋めます。
